Сформируй техническое задание на доработку системы управления мастер данных, оформи в соответствии с правилами написания ТЗ на разработку::
Для справочника "Классификатор ТРУ" необходимо реализовать поддержку управления версиями, статусами и сроками действия справочников, аналогичную реализованной для справочника «Структура ПБГ»
В справочник "Классификатор ТРУ" необходимо добавить следующие поля со следующей логикой (типы данных и прочая информация указаны в ПР 1968 Классификатор ТРУ):
1.	Поле CLASS_LEVEL (Уровень класса). Формируется системой автоматически по правилу: Значение, отражающее уровень данного класса в иерархии.
•	Уровню 1 советует корневая запись "Статус классификатора"
•	Уровню 2 соответствует "Дата начала действия структуры ТРУ"
•	Уровню 3 - "Бизнес-сегмент"
•	Уровню 4 - "Функциональное направление ТРУ"
•	Уровню 5 - "Область"
•	Уровню 6 - "Категория ТРУ"  - является последним допустимым в иерархии ТРУ. Одна запись не может относится одновременно к нескольким уровням иерархии.
2.	Поле TRU_STATUS (Статус). Заполняется текстом из поля "Статус" (STATUS/STATUS) той записи уровня "Статус классификатора", которая является  предком для элемента классификатора. Обязательно для заполнения для Class_level = "Статус классификатора". Варианты значений: [‘01. Архив’,’ 02. Текущая структура’,’ 03. Будущая структура’]
3.	Поле DATE_BEGIN (Дата начала действия структуры ТРУ). Заполняется значением поля DATE/DATE_BEGIN уровня "Дата начала действия структуры ТРУ". Обязательно для заполнения для Class_level = "Дата начала действия структуры ТРУ".
4.	Поле DATE_END (Дата окончания действия структуры ТРУ). Заполняется значением из поля DATE/DATE_END уровня "Дата начала действия структуры ТРУ". Обязательно для заполнения для Class_level = "Дата начала действия структуры ТРУ".
5.	Поле PREV_CLASS_REF (Соответствие классу предыдущей версии ТРУ). Заполняется кодом КССС класса ТРУ предыдущей версии справочника ТРУ. Данное поле необходимо для реализации связи записей текущего перечня предшествующему. Для ведения данного поля в интерфейсе карточки класса ТРУ необходимо добавить возможность внесения пользователем данной связи. Поле должно поддерживать «Множественность».
